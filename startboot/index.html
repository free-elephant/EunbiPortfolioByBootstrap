프로젝트 기획 및 개발



웹 서비스 기획 및 개발


1. 상담 커뮤니티 '소근소근'

- 서비스 설명: 누구나 고민을 말하고 들어줄 수 있는 커뮤니티 서비스
- 역할: 기획, UI UX 디자인, 프론트엔드
- 처음으로 여러 개발자와 협업했다. 중심적인 기능만 기획한 채 만들다 보니 페이지마다 폰트 크기도 달랐고, 한 페이지를 만들 때마다 ‘이 버튼 클릭하면 뭘 띄우지?’ 등의 질문이 나오며 프로젝트 기간이 길어졌다. 요구 사항 정의가 중요하다는 것을 배울 수 있었다.
- 구현 화면을 보고 싶다면 클릭해 주세요:D

2. P2P 택배 서비스 '퍼플'

- 서비스 설명: 물건을 보내고 싶은 개인과 물건을 배달하고 싶은 개인을 연결해 주는 P2P 택배 서비스 플랫폼
- 역할: 기획, 풀스택
- 구현 기능을 정리한 후, 예상 구현 시간을 계획한 채 프로젝트 진행했다. 각자 파트를 맡아 진행했는데, 개발자마다 개발 속도가 달랐고 오류를 처리하다 보니 예상 시간을 훌쩍 넘기게 되었다. 프로젝트를 관리할 때 각 개발자의 능력과 오류 상황까지 고려하여 계획해야 한다는 것을 배웠다.
- 구현 화면을 보고 싶다면 클릭해 주세요:D

3. 자율 주행 카트

- 서비스 설명: 카트 앞 카메라를 설치해 사람의 뒷모습을 인식하여 따라가게 하는 IOT 프로젝트 진행
- 역할: 프로젝트 전체 관리 및 카트 속도 조절 코딩(Tracking 오픈소스 변형)
- 카트를 만드는 팀과 Object Tracking 코드를 짜는 팀으로 나누었다. IOT도 처음이었고, 이미지를 학습시키는 것도 처음이라 총 3주 정도의 프로젝트 시간 동안 2주 동안 코드를 고민했다. 너무 생소한 분야라 포기하고 싶을 때도 있었는데, 구글 선생님의 도움으로 결국 해냈다. 역시 끝까지 하는 게 중요하다.
- 구현 화면을 보고 싶다면 클릭해 주세요:D

회사 제품 소개 페이지 제작


- 회사 소개: 회사에서 프로젝트를 진행할 때, 비대면으로 할 수 있게 도와주는 협업 툴 제작 회사
- 역할: 프론트엔드
- 회사소개 페이지를 제작했다. 페이지를 기획한 기획자 및 개발자분들과 협업했다. 기획자가 자신의 요구 사항을 개발자에게 어떻게 정리하는지를 볼 수 있었다. 해당 페이지의 목적에 따라, 기획한 기능을 버리기도 하고 덧붙이기도 했다. IT 회사의 생태계를 단편적으로나마 볼 수 있었던 시간이었다.
- 구현 화면을 보고 싶다면 클릭해 주세요:D
